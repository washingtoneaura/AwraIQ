Awra System - Login

Overview

The Awra Inventory & Procurement System is a multi-tenant web-based platform designed to manage inventory, procurement processes, and vendor quotations with AI-powered analysis. It supports advanced features for notifications, auditing, and real-time monitoring.

Installation

git clone https://github.com/yourusername/awra-inventory-procurement.git
cd awra-inventory-procurement
composer install
cp .env.example .env
php artisan key:generate
php artisan migrate --seed
php artisan serve

System Architecture

The system is built using Laravel with:

  • MySQL Database
  • Redis caching
  • Queue jobs for async tasks
  • AI-powered Python service for quotation analysis
  • Frontend with Blade templates, Tailwind CSS, and Alpine.js

Features

  • Multi-tenant inventory management
  • Dynamic role-based access control
  • Item, category, vendor, and quotation management
  • AI-powered procurement quotation evaluation
  • Automated notifications and email logs
  • Integration with MPESA and Mail services
  • Monitoring with Laravel Horizon, Pulse, and Telescope

Multi-Tenant Setup

All data is associated with tenants using custom_id. Key tables include:

  • tenants
  • users
  • items
  • vendors
  • quotations

Tenant isolation ensures secure and separate data per organization.

Roles & Permissions

Users are assigned roles via role_user table. Roles define access to:

  • Items management
  • Vendor and procurement features
  • Administrative dashboards

Items Management

Items can be added, edited, and deleted. Important rules:

  • Unique item IDs generated per day
  • Cannot delete items with pending adjustments or unapproved quotations
  • Supports photo upload and custom fields per tenant

Vendors & Procurement

Procurement requests (RFQs) and quotations are managed with:

  • Vendor selection
  • Quotation submission
  • Approval workflow
  • Purchase order generation
  • Order fulfillment tracking

AI-Powered Quotation Analysis

The system evaluates vendor quotations using:

  • Item-wise price comparison
  • Overall quotation scoring
  • Delivery time and vendor performance metrics

Notifications & Logging

System uses:

  • Email notifications (Mailgun / Mailjet)
  • SMS notifications
  • Audit logs and activity tracking
  • Telescope for monitoring requests

API Endpoints

GET /api/items
POST /api/items
GET /api/quotations
POST /api/quotations
GET /api/vendors
POST /api/vendors

Monitoring: Horizon / Pulse / Telescope

  • Horizon: queues and job monitoring
  • Pulse: app performance metrics
  • Telescope: request inspection and logs

FAQ

  • How do I reset migrations? Use php artisan migrate:fresh --seed.
  • Who can access Pulse? Only authorized emails (configurable in Pulse auth closure).
  • How do I add new tenants? Via the admin panel under tenant management.
